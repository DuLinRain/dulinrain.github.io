JS中的console API介绍

在大多数JavaScript引擎中，都有一个带有日志打印和调试方法的全局对象console。 该对象不属于JS语言本身，但自从被Firebug调试器开创以来，已经成为了事实上的标准。由于它的主要目的是调试，console方法在开发过程中最常使用，很少在已部署的代码中使用。

这篇文章将对console API以及它在不同浏览器下的兼容进行讲解。

一、console API跨浏览器兼容性如何？

Firebug首先提供了控制台console API，其wiki中的文档与标准最为接近。 此外，Brian Kardell和Paul Irish正在制定API规范，这将在跨浏览器方面产生更加一致的行为。在标准制定完成之前，浏览器的行为差异还是会很大。因此，这篇文章只是提供一个概述，想了解更加详细的信息，请参考各种平台的文档。

二、简单用法

  • console.clear()

清除控制台

  • console.debug(object1, object2?, ...)

建议优先使用console.log，和这个是一样的效果

  • console.error(object1, object2?, ...)

将参数打印到控制台。 在浏览器中，记录的内容可能会标记为“error”图标和/或包含堆栈跟踪或代码链接。

  • console.exception(errorObject, object1?, ...])Firebug独有

打印对象，并且还有堆栈跟踪信息

  • console.info(object1?, object2?, ...)

和console.log类似

  • console.log(object1?, object2?, ...)

将参数记录到控制台。如果第一个参数是printf样式的格式字符串，则使用它来打印剩余的参数。 示例（Node.js REPL）

唯一可靠的跨平台格式化指令是％s。 Node.js支持％j将数据格式化为JSON，浏览器倾向于支持将某些内容交互记录到控制台的指令。

  • console.trace()

打印堆栈跟踪信息

  • console.warn(object1?, object2?, ...)

同console.error，只不过图标会是警告

三、检查和计数

  • console.assert(expr, obj?)

类似于其它语言中的断言，如果expr的值是false, 打印obj，否则啥也不干

  • console.count(label?)

用此标签计算执行此语句的行的次数。

四、格式化打印

  • console.dir(object)

将对象的表示形式打印到控制台。 在浏览器中，可以交互式地查看（在Node.js中，不能）。其实效果和console.log(object)一样

  • console.dirxml(object)

打印HTML或XML的树

  • console.group(object1?, object2?, ...)

将对象记录到控制台并打开一个包含所有将来记录的内容的嵌套块。 该块通过调用console.groupEnd（）来关闭。 该块最初是展开的，但可以折叠。

  • console.groupCollapsed(object1?, object2?, ...)

同console.group，差别一个是展开一个是折叠。

  • console.groupEnd()

用来结束console.group、console.groupCollapsed

  • console.table(data, columns?)

以数组形式打印数组，每行一个元素。 可选参数列指定哪些属性/数组索引显示在列中。 如果缺少该参数，则将所有属性键用作表格列。 缺少的属性和数组元素在列中显示为未定义。

四、性能和计时

  • console.markTimeline(label) Safari独有

同timeStamp

  • console.profile(title?)

打开性能分析。 可选参数标题用于分析报告。

  • console.profileEnd()

结束性能分析。 打印分析报告。

  • console.time(label)

开启一个标签叫做label的计时器

  • console.timeEnd(label)

结束一个标签叫做label的计时器

  • console.timeStamp(label?)

用给定的标签记录时间戳。 可以记录到控制台或时间线上。
