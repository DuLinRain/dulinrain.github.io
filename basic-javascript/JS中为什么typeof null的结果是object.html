JS中为什么typeof null的结果是object?

写JS的人都知道，JS中typeof null=== 'object'，这是JS的基本知识，也是一些考试/面试中经常会被问到的问题。几乎很少有人去追寻为什么是这个结果。因为按照我们的知识，null不是一个object，它属于JS中的基本数据类型。

其实，typeof null=== 'object'是JS中的一个bug!!，并且非常不幸，这个bug可能永远也不会被修复了。因为修复这个bug可能会导致很多现有的代码无法正常工作。下面我们来看看这个bug产生的历史。

一、typeof null的前世今生

“typeof null” bug是JS第一个版本的历史遗留问题。在这个版本中，JS中的值存储在32bit中，其中低1~3bit用来表示数据类型，剩余的bit用来表示真实的值。3bit可以用来表示8中类型，但是呢，实际上这个版本只定义了5种类型：

  • 000: object. 表示引用类型Objetc.
  • 1: int. 表示31位有符号整数。
  • 010: double. 浮点数.
  • 100: string. 字符串.
  • 110: boolean. 布尔型。

也就是说，最低为要么是0要么是1。最低位是1的话就意味着，类型只占1比特，也就是int型。最低位是0的话，意味着类型占3bit，除去最低位，用另外2位表示四种类型，也就是object，double，string，boolean。

有两个值是特殊值：

  • undefined (JSVAL_VOID) 表示整数-230。这个数超过了当时整数的表示范围。
  • null (JSVAL_NULL) 表示机器码NULL指针，或者说是一个指向0的对象引用。

这就很明显，为什么typeof null会得到object的结果了： 它检查null的类型，结果类型标识告诉它说结果是object。

下面就是typeof的机器代码实现：

JS_PUBLIC_API(JSType)
JS_TypeOfValue(JSContext *cx, jsval v)
{
    JSType type = JSTYPE_VOID;
    JSObject *obj;
    JSObjectOps *ops;
    JSClass *clasp;

    CHECK_REQUEST(cx);
    if (JSVAL_IS_VOID(v)) {  // (1)
        type = JSTYPE_VOID;
    } else if (JSVAL_IS_OBJECT(v)) {  // (2)
        obj = JSVAL_TO_OBJECT(v);
        if (obj &&
            (ops = obj->map->ops,
             ops == &js_ObjectOps
             ? (clasp = OBJ_GET_CLASS(cx, obj),
                clasp->call || clasp == &js_FunctionClass) // (3,4)
             : ops->call != 0)) {  // (3)
            type = JSTYPE_FUNCTION;
        } else {
            type = JSTYPE_OBJECT;
        }
    } else if (JSVAL_IS_NUMBER(v)) {
        type = JSTYPE_NUMBER;
    } else if (JSVAL_IS_STRING(v)) {
        type = JSTYPE_STRING;
    } else if (JSVAL_IS_BOOLEAN(v)) {
        type = JSTYPE_BOOLEAN;
    }
    return type;
}

上面代码的运行步鄹如下：

  • 在（1）处，代码首先检查v是不是undefined, 这个检查是通过==判断的：

#define JSVAL_IS_VOID(v)  ((v) == JSVAL_VOID)

  • 然后（2）处用来检查v是不是object类型。如果它是可调用的(3)或者内部有 [[Class]]属性，那么它认为v是函数，否则的话类型就是object。这就是typeof null == 'object'的由来
  • 接下来就是对 number, string 和 boolean 类型的检查，整个代码没有一处明确的用来检查null的代码。其实要实现的话可以这么实现：

#define JSVAL_IS_NULL(v)  ((v) == JSVAL_NULL)

这个bug现在看起来是一个非常明显的bug，但是考虑到第一版JS只花了很短很短的时间就完成了，所以长得那么帅，我们能怎么滴，还不是选择原谅它啦。
