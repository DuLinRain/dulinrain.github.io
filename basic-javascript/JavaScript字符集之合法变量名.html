二、ES5之合法变量名

2.1、合法变量名定义

ES5方案中是这么说的：

An Identifier is an IdentifierName that is not a ReservedWord

也就是说标识符（变量名）是一个标识符名称，这个标识符名称不能是保留字。 越说越复杂，反而引入了更加生僻的专有名词。通过这句话，我们可以大概知道有这么几个概念：

  1. 标识符（Identifier）
  2. 标识符名称（IdentifierName）
  3. 保留字（ReservedWord）
  4. 标识符是标识符名称的一个子集

下面我们对这几个概念分别解释，力求循序渐进：

保留字

JS中，保留字分为4种：关键字（keywords）、未来保留字( future reserved words)、null字和Boolean字。

  1. 关键字。关键字在JS中有特殊的意义，通常用来表示变量声明、属性操作、流程控制等。有一下这些： break, case, catch, continue, debugger, default, delete, do, else, finally, for, function, if, in, instanceof, new, return, switch, this, throw, try, typeof, var, void, while, and with.
  2. 未来保留字。未来保留字表示这些字符可能在JS的未来版本中成为关键字。他们有： class, const, enum, export, extends, import, and super。 一些未来保留字只能在严格模式下使用： implements, interface, let, package, private, protected, public, static, yield。
  3. null字。null字就是指null.
  4. Boolean字。Boolean字就是指true和false.

注意：另外，还有一些字符，虽然他们不是保留字，但是他们表现的却像是关键字，比如说:NaN ,Infinity ,undefined。这些字符都是全局对象的属性，而且是不可更改的（immutable）属性，所以，尽管在全局空间，我们可以声明类式var NaN=25这样的变量，但是却起不到任何效果。举个栗子：

另外，在更早的ES3标准中规定的一些保留字，在ES5中，不再被规定为保留字。这些字符有： int, byte, char, goto, long, final, float, short, double, native, throws, boolean, abstract, volatile, transient,synchronized.（看了这些感觉JS在强类型语言和弱类型语言之间强烈的摇摆啊！）。

当然，为了尽量保持代码的后向兼容性，我们在声明变量时，最好不要使用这些字符。

标识符名称

标识符名称的完整定义比较复杂，我们仅仅提取常常用到的那些方面做一个定义： 标识符名称必须以美元符（$）或下划线（_）或Unicode字符等打头，以这些符号作为后继。注意：定义并没有规定标识符名称不允许使用保留字。

标识符

有了上述讲解，那么很好理解标识符的定义了：

An Identifier is an IdentifierName that is not a Reserved Word

标识符是不属于保留字的标识符名称。

标识符是标识符名称的子集
