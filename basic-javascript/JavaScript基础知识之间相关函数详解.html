我们知道，当传递给SetTimeout的第一个参数是函数名的时候，我们是可以给它传递参数的。比如：

function A(x){
  console.log(x);
}
setTimeout(A(1),10);//1

但是，当第一个参数是一个匿名函数时，如果浏览器不支持第一种语法，我们是没办法给它传递参数的，如下：

setTimeout(function(args){
  //do somrthing
},10);

我们是没有办法给这个匿名函数传递参数，那么为了让不支持第一种语法的浏览器也支持参数传递，我们该怎么办呢？

六、setTimeout和setIntervalthis问题

6.1、this问题

setTimeout和setInterval中的this是存在问题的，如下：

myArray = ['zero', 'one', 'two'];
myArray.myMethod = function (sProperty) {
    alert(arguments.length > 0 ? this[sProperty] : this);
};

myArray.myMethod(); // prints "zero,one,two"
myArray.myMethod(1); // prints "one"

但是，如果在setTimeout和setInterval中调用，则会出现一些问题，如下：

setTimeout(myArray.myMethod, 1000); // prints "[object Window]" after 1 second
setTimeout(myArray.myMethod, 1500, '1'); // prints "undefined" after 1.5 seconds

myArray.myMethod被传递给setTimeout函数，当myArray.myMethod被调用时，它的this没有被设定，所以this默认指向了window。有人想到了call,apply等，想着是不是可以给它传一个东西让this指向它呢？答案是不行。如下：

setTimeout.call(myArray, myArray.myMethod, 2000); 
// error: "NS_ERROR_XPC_BAD_OP_ON_WN_PROTO: 
//Illegal operation on WrappedNative prototype object"
setTimeout.call(myArray, myArray.myMethod, 2500, 2); // same error

6.2、this问题的解决方法

6.2.1、用匿名函数包裹

可以用一个匿名函数包裹，然后在匿名函数里面调用

setTimeout(function(){myArray.myMethod()}, 2000); 
// prints "zero,one,two" after 2 seconds
setTimeout(function(){myArray.myMethod('1')}, 2500); 
// prints "one" after 2.5 seconds
6.2.2、使用箭头函数包裹
setTimeout(() => {myArray.myMethod()}, 2000); 
// prints "zero,one,two" after 2 seconds
setTimeout(() => {myArray.myMethod('1')}, 2500); 
// prints "one" after 2.5 seconds

七、为什么会比指定的延时更长？

有时候，指定一个较小的延时，实际回调函数的调用时间比指定的延时时间长，这是为什么呢？有很多原因，但是最主要的原因是浏览器在实施setTimeout/setInterval的时候做了一定的限制：

  1. 在HTML5之前的版本，最小限制是15.6ms（也有说是10ms）
  2. 在HTML5版本，最小限制是4ms
  3. 以上只是针对活动标签，对于非活动标签页，不同浏览器时间限制不同，有1000ms、10000ms等
  4. 在有些浏览器中，如果非活动标签页存在audio，那么该页面的时间将不受第3条限制。

八、为什么是15.6ms或4ms?

这是根据硬件每秒钟可以响应的中断数来算的。

  1. 在HTML5之前的版本，认为（或默认）计算机每秒钟可以响应64个中断，所以1000ms/64=15.6ms。
  2. 而在HTML5中，认为（或默认）计算机每秒钟可以响应250个中断，所以1000ms/250=4ms。
  3. 而对于10ms的解释，暂时不清楚。

九、setTimeout(callback,0)、setInterval(callback,0)

前面说了，setInterval必须指定第二个参数，如果第二个参数delay小于10则强制置成10. setTimeout可以不指定第二个参数，如果不指定delay，则默认设置为0，表示立即执行或“尽快执行”。为什么说是“尽快执行”呢？因为实际上这里的“立即执行”是有水分的，或者说是指立即执行“把callback加入到事件队列”这么个事。只有当队列前面的都执行完了，才可以执行它。

看看英文解释：

This is because even though setTimeout was called with a delay of zero, it's placed on a queue and scheduled to run at the next opportunity; not immediately. Currently-executing code must complete before functions on the queue are executed, thus the resulting execution order may not be as expected.
