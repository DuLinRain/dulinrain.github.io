HTTP状态码中301含义及其缓存策略

一、概述

我们在HTTP状态码中302和307有什么区别？以及HTTP状态码中303与302、307有什么区别？中已经讲过了302(Found)、307(Temporary Redirection)、303(See Other)这几种状态码的含义以及他们之前的相同点和不同点，今天我们继续看看HTTP状态码301的含义。

二、 301

HTTP中301的含义就是我们通常所谓的“永久重定向”, 通常我们认为是与307临时冲顶向相对应的一个码。但是，实际上这个状态码应该叫“永久移动（Moved Permanently）”。

从以上描述我们可以看到，它和302的唯一区别就是它的含义表示“永久移动”，它也存在和302一样的问题就是客户端通常会把redirect请求用GET方式，而不是按照original请求的方式。这一点我们可以从下面这个例子可以看出：

服务端代码：

var express = require('express');
var cors = require('cors')
var app = express();
app.use(cors())

app.all('/test', function (req, res) {
  console.log('original请求方式: ' + req.method)
  res.set({
    // 'Cache-Control': 'max-age=3600',
    'Location': 'http://127.0.0.1:3000/test1',
  }).status(301).end('hehe1')
});
app.all('/test1', function (req, res) {
  console.log('redirect请求方式: ' + req.method)
  res.set({
    // 'Cache-Control': 'max-age=3600'
  }).end('redirect请求')
});

var server = app.listen(3000, function () {
  var host = server.address().address;
  var port = server.address().port;

  console.log('Example app listening at http://%s:%s', host, port);
});

三、缓存策略

HTTP协议规定，301是可缓存的。协议中并没有说必须要指定了特定的头域才能被缓存。这一点和302、307还有有很大区别的。

也就是说，只要不刻意指定不缓存的头域，301默认都是会被缓存的。而302/307则是，默认不缓存，除非用Cache-Control或Expires头域指定缓存。

我们来证明一下：

服务端代码：

var express = require('express');
var cors = require('cors')
var app = express();
app.use(cors())

app.all('/test', function (req, res) {
  console.log('original请求方式: ' + req.method)
  res.set({
    // 'Cache-Control': 'max-age=3600',
    'Location': 'http://127.0.0.1:3000/test1',
  }).status(301).end('hehe1')
});
app.all('/test1', function (req, res) {
  console.log('redirect请求方式: ' + req.method)
  res.set({
    // 'Cache-Control': 'max-age=3600'
  }).end('redirect请求')
});

var server = app.listen(3000, function () {
  var host = server.address().address;
  var port = server.address().port;

  console.log('Example app listening at http://%s:%s', host, port);
});

客户端代码：

<!DOCTYPE html>
<html>
<head>
    <title>httptest client</title>
    <script src="https://unpkg.com/axios/dist/axios.min.js"></script>
    <!-- <script src="./qs.js"></script> -->
</head>
<body>
    <button id="test">测试按钮</button>
    <script>
        document.querySelector('#test').addEventListener('click', function () {
          axios.get('http://127.0.0.1:3000/test', JSON.stringify({}))
          .then(function (response) {
            console.log(response);
          })
          .catch(function (error) {
            console.log(error);
          });
        })
    </script>
</body>
</html>

注意：我们将客户端的代码改成了GET请求，因为如果改成POST，不管服务端设不设置Cache-Control，original请求都不会被缓存。

四、总结

301和302含义是有区别的，但是浏览器对它们的行为是一致的，浏览器对他们的缓存策略不一致。
