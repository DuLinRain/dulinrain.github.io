HTTP状态码中302和307有什么区别？

一、 概述

在HTTP状态码中，3XX表示浏览器需要采取进一步的行动以便完成这次请求。而这其中我们最为常见的是重定向。说到重定向就不得不说一说302。一般我们都把302称为“临时重定向”。但其实302并不是临时重定向。302真实的名字叫Found。HTTP确实有一个叫做“临时重定向”的状态码，这个状态码是307。表示那么302和307各表示什么呢？为什么我们常常会把302认为是临时重定向呢？他们有什么相同点和不同点呢？

二、302与307

实际上，翻看HTTP协议（RFC2616），我们会惊奇的发现，302和307几乎可以说是完全一样的啊，可以说一字不差啊。我们把协议贴出来对比一下：

但是呢，其实RFC2616在描述302的时候还有一小段注解如下：

这段话什么意思呢？其实说的就是RFC1945(Hypertext Transfer Protocol -- HTTP/1.0, 也就是HTTP/1.0提案(PROPOSED STANDARD))和RFC2068（Hypertext Transfer Protocol -- HTTP/1.1，也就是HTTP/1.1的提案(PROPOSED STANDARD)）中有明确规定，客户端不允许改变原始的请求类型，但是很多客户端的实现都没按照这个规定来。

简单来说就是，假如客户端发出请求A后收到302了，按规定来说，客户端需要以与A相同的请求类型去请求302响应头中Location指定的资源，不允许改变类型。然而很多客户端的实现都不按照这个走，都是无论A的请求类型是什么，客户端收到302后都会以GET方式去请求Location中指定的资源。

而这种"不论初始请求类型，都按GET请求Location资源"的方式恰好与协议对303的规定行为相同。

307则不会出现这种不按协议走的情况。

这样就可以明确区分出302和307了。

三、实例验证

为了验证前面的说法，我们用两个实际的例子来对比一下。首先我们有一个node的server端代码如下：

  console.log('original请求方式: ' + req.method)
  res.set({
    'Location': 'http://127.0.0.1:3000/test1'

});
app.all('/test1', function (req, res) {

  var host = server.address().address;

四、总结

总结一下就是这几点：

  • 302和307实质内容是一样的，只是客户端对他们的处理方式不一样。
  • 客户端收到302后会以GET方式请求Location中的资源
  • 客户端收到307后会以前面发出去的那个请求方式去请求Location中的资源，如果是发的GET得到的307,则客户端以GET继续请求Location中的资源，如果是发的POST得到的307,则客户端以POST继续请求Location中的资源，以此类推。
