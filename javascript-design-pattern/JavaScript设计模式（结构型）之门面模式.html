JavaScript设计模式（结构型）之门面模式（facade）

一、概述

门面模式也称外观模式，是一种经常用到也非常简单的模式。门面模式一般用来为一些实现相似功能的接口提供一个统一的访问入口。

二、门面模式

上一章学习了适配器模式，里面有个思想是，旧API是一个将计算字节长度和字符长度合在一起的接口，然后适配器模式是，将这两个功能拆分出来，用适配器提供统一的访问入口，然后将适配器映射到旧API。

门面模式的场景与这有很相似，但又有明显不同。

  1. 门面模式没有所谓的旧API概念，门面模式应用之前，各内容都是一个互相独立的功能。
  2. 门面模式提供统一的接口，但不会把这个接口映射到所谓的“旧API”
