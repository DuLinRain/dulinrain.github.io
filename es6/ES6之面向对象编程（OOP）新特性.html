ES6之面向对象编程（OOP）新特性

一、概述

Class是ES6中最主要的面向对象编程新特性，除此之外，对象字面量以及一些实体方法也拥有面向对象新特性。

1.1、方法定义

ES5在对象字面量中定义方法通常采用如下语法：

let obj = {
  myMethod: function(){
    .....
  }
}

在ES6中可以采用这种方法：

let obj = {
  myMethod(){
    .....
  }
}

是不是简单方便很多？

1.2、属性值速记

传统方法：

let first = 'lin'
let last = 'yu'
let obj = {
  first: first,
  last: last
}

ES6中，当属性字和属性名一样时，可以采用速记法：

let obj = {first, last}

二、对象字面量中新特性

2.1、方法定义

在ES5中，对象字面量中的方法其实就是一个属性，只不过这个属性的值是一个function:

let obj = {
  myMethod: function(){
    .....
  }
}

在ES6中，对象字面量中的方法仍然是值为function的属性，但是有一个更紧凑的写法：

let obj = {
  myMethod(){
    .....
  }
}

getter和setter仍然起作用，就像它们在ES5中一样（注意它们的语法和方法定义非常像）。

对于属性值是generator的属性，也存在着一种精简的定义方法：

let obj = {
  * myGenerator() {
   ...
    }
}

等效于：

let obj = {
  * myGenerator: function() {
   ...
    }
}

2.2、属性值速记

如果作为属性值得变量的名称和属性key的名字一样，那么你可以省略属性key。如下：

let first = 'lin'
let last = 'yu'
let obj = {
  first: first,
  last: last
}

等效于：

let obj = {first, last}

这个与结构的结合也非常好：

let {first, last} = {first: 'lin', last: 'yu'}

三、Object上的新方法

3.1、Object.assign(target, source1, source2,....)

Object.assing方法将N个source合并到target对象。该方法通过两个步骤来修改target:

  1. 将所有后面source的可枚举自有属性（非继承）拷贝出来
  2. 将这些属性添加到target上

让我们仔细看看Object.assign()是如何工作的：

  1. 支持string和Symbol作为key。Object.assign()同时支持string和Symbol作为key
  2. 只拷贝可枚举自有属性。Object.assign()将忽略继承而来的属性以及不可枚举属性
  3. 通过赋值（assignment）来完成拷贝。target中的属性是通过赋值（内部属性[[put]]）创建的。这意味着，如果target本身有（不管是自身的还是继承的）setter，那些setter在赋值的时候将会被调用。另外一种定义属性的方法是define，该方法永远创建一个新的只有属性，永远不会调用setter。在ES6里面本来有个提案是将define作为Object.assign()的实现，但是该草案被回绝了。
  4. 如果一个方法使用了super,那么你将无法移动该方法。如果一个方法使用了super，那么该方法中将会有一个内部属性[[HomeObject]]将它绑定到创建它的对象，如果你通过Object.assign移动它，它将会仍然指向最初的对象。

3.1.1、Object.assign的使用案例——给this增加属性

你可以在构造函数中通过Object.assign为this对象增加属性：

class A{
    constructor(x,y){
        Object.assign(this,{x, y})
    }   
}

3.1.2、Object.assign的使用案例——给对象属性提供默认值

Object.assign也可以用来给对象属性提供默认值：

const DEFAULTS = {
    level: 0,
    form: 'html'
}
function processContent(options){
    Object.assign({}, DEFAULTS, options) //（A）
}

在A行中，我们创建了一个新的空对象{}，然后将DEFAULTS拷贝给它，然后将options拷贝过去覆盖上一步的默认值。

3.1.3、Object.assign的使用案例——给对象增加方法

另一种使用情况是给对象增加方法：

Object.assign(someClass.prototype, {
    methodA() {
        ...
    },
    methodB() {
        ...
    }
})

这个相较于下面这个版本简单多了：

someClass.prototype.methodA = function(){
    ...
}
someClass.prototype.methodB = function(){
    ...
}

3.1.4、Object.assign的使用案例——克隆对象

Object.assign的一个用法是快速的克隆对象：

Object.assign（{}， orgobj）

注意，这种克隆方法只会克隆orgobj的可枚举自有属性，对于继承而来的属性以及不可枚举属性是克隆不出来的，也就是说它不会维持原型链，这种克隆方法常常被称为“浅复制”或“脏复制”

那么，要想实现深复制怎么办呢？

我们需要结合Object.create()和Object.assing()来实现：

  1. 首先用Object.getPrototypeOf()拿到原型对象
  2. 其次用Object.create()实现继承
  3. 然后将自有可枚举属性拷贝到上一步得到的对象上

3.2、Object.getOwnProtetypeSymbol(obj)

Object.getOwnProtetypeSymbol(obj)获取所有key为Symbol的属性，它是Object.getOwnPrototypeNames(obj)的补充（Object.getOwnPrototypeNames(obj)只能获取key为string的属性）

3.3、Object.is(obj1,obj2)

提供了一个比===更为精确的方法用于判断两个值是否相等，实际上就是实现了NaN===NaN以及+0===-0的判断。

注意[]仍然不等于[],{}仍然不能于{}，两个长得一样的数组和对象还是不会被认为相等。

3.4、Object.setPrototypeOf(obj,proto)

Object.setPrototypeOf(obj,prpto)用来将obj的原型设置为proto。其实这个方法实现的就是ES5中的一个非标准但又被很多浏览器引擎支持的方法：obj.__proto__ = proto。ES5中推荐的设置原型的方法仍然是在创建对象的时候，这远比先创建一个对象obj，再设置proto快很多：

let obj = Object.create(proto)

什么意思呢？也就是说let obj = Object.create(proto)创建了一个obj对象，并且指定了它的原型：

let obj ={}
obj.__proto__ = proto

四、ES6中对属性key的迭代

ES6中属性的key既可以是string,也可以是symbol。现在你有5种方法可以获取一个属性的key:

  1. Object.keys(obj): Array。枚举所有key为string的可枚举自有属性
  2. Object.getOwnPrototypeNames(obj): Array。枚举所有key为string的自有属性。（也就是说还可以枚举不可枚举属性？）
  3. Object.getOwnPrototypeSymbols(obj): Array。枚举所有key为symbol的自有属性。（也就是说还可以枚举不可枚举属性？）
  4. Object.ownKeys(obj): Array<string|symbol>。枚举所有属性，不管key是string还是symbol。（也就是说还可以枚举不可枚举属性？）
  5. Reflect.enumerate(obj): Iterator。枚举所有key为string的可枚举属性

4.1、属性的key的迭代顺序

所有的迭代方法都按如下顺序迭代：

  1. 首先迭代数组索引，按数值排序
  2. 其次迭代key为string的属性，按照他们创建的顺序
  3. 最后是symbol，按照他们创建的顺序

注意，ES标准对string索引的定义如下：如果一个数值（索引）转换成unsigned int然后再转换回去之后，该数值的string和转换前数值的string没有任何变化，那么它就是string索引。什么意思呢？

我们知道数值是可以作为对象的key的，这是因为实际上它在里面被视为了string。比如：

let obj = {
    0: 100
}

按照上面关于string索引的定义，因为0转为无符号整数是0，再转回去仍然是0，所以他其实被认为是string,也就是真正的key变成了‘0’。
