五、模拟命名参数

当你调用一个函数或者方法的时候，你需要映射实参到形参，有两种普遍的形式来干这个事儿：

  1. 按位置映射。第一个实参映射给第一个形参，第二个实参映射给第二个形参，以此类推。
  2. 命名参数使用名字来完成映射。名字是与函数定义时的形参相关联并且与函数调用时的实参的标签相对应，调用者不必关心参数的传递顺序。

命名参数有两个主要的优点：

  1. 它们在函数调用时对参数进行描述
  2. 它们对于可选参数工作的很好。

下面先介绍命名参数的优点，然后介绍如何在js中通过对象字面量来模拟。

5.1、命名参数作为描述

当函数的参数有很多个的时候，很容易让人迷惑到底哪个参数代表什么意思。比如下面这个函数调用：

test(1,20,2)

python语言支持命名参数，它的调用方式如下：

test(start=1, end=20, step=2)

5.2、可选命名参数

可选固定位置参数只可以省略最后的参数，也就是说如果后一个参数没有省略，那么前面也不允许有省略的参数。

但是，如果支持命名参数，那么任意位置的参数都是可以省略的，比如python语法支持如下：

test(start=1)
test(end=20, step=3)
......

5.3、在JS中模拟命名参数

JS不想python等语言那样支持原生的命名参数，但是它有一种非常优雅的方式用对象字面量来实现命名参数：

test({start: 1, end: 20, step: 2})

这个函数接收一个对象，该对象拥有start,end,step三个属性，你可以省略它们中的任意：

test({start: 1, end: 20, step: 2})
test({end: 20, step: 2})
test({end: 20})
test()

在ES5中，test的实现通常长下面这个样子：

function test(options){
  options = options || {}
  var start = options.start || 0
  var end = options.end || 20
  var step = options.step || 2
}

在ES6中就不用这么麻烦了，你可以使用解构：

function test({start = 1, end = 20, step = 2}){
  .......
}

八、展开运算符（...）

8.1、函数调用以及方法调用中的展开运算符

展开运算符与剩余运算符长得一模一样，但是它们正好相反：

  1. 剩余运算符用来提取数组中的数据，用于剩余参数以及解构
  2. 展开运算用于在函数调用时将数组转换为参数，也用于将数组转换为数组字面量中的元素

比如，Math.max可以接受任意个参数，但是不能接收数组，用展开运算符就可以做到：

Math.max(...[1,2,3])

与剩余运算符不同，展开运算符可用在任意位置，不限定为最后参数：

Math.max(1，2，...[1,2,3]，3，4，5)

另一个用法是push:

arr1.push(...[1,2,3,4])

8.2、构造函数中的展开运算符

new Date(...[2017,07,07])

8.3、展开为数组

[1,2,...[1,2,3,4],5,6]
得到
[1,2,1,2,3,4,5,6]

则可以很方便的用来实现连接数组：

[...[1,2,3],...[4,5,6],...[7,8,9]]
得到
[1,2,3,4,5,6,7,8,9]
