ES6之箭头函数

一、概况

ES6提供了一种新的函数--箭头函数，箭头函数可以带来两大好处：

  1. 相比于传统函数，它们更简洁

  2. 箭头函数的this从箭头函数的外围得到，也就是说具有词法性质（lexical）,因为你不再需要使用bind或者that=this这样的变通（workaround）

在ES6箭头函数中的一下变量都是lexical的：

  1. this
  2. super
  3. arguments
  4. new.target

由于super和new.target都有其特定的使用场合，这里不举例子证明他们的lexical了。

二、传统函数是不好的“非方法函数（non-method function）”，就是因为this的缘故！

2.1、ES5中this存在的问题

在JavaScript里面，传统函数可以以一下几种形式调用：

  1. 非方法函数（non-method function）
  2. 方法（methods）
  3. 构造函数（constructors）
这里先说一下非方法函数，“非方法函数”一般是指普通函数表达式、函数声明和回调函数。方法一般是指定义在对象字面量或者类中的。function和method还是有区别的！！

上述三种角色实际上是有冲突的：角色2和角色3函数内部都会有自己的this,但是当以角色3被调用时（比如回调函数），这个this会阻止你拿到该函数外围函数的this。这就是概况中所讲的例子，回调函数中的this指向的是button dom对象，而不是UiComponents函数的this。

注意：不同的调用方式，this指向不一样，这个可以参考“可调用实体”那一章节

4.2、箭头函数的一些变量都是lexical的

箭头函数和传统函数的一个重要区别在于：

  1. 箭头函数的this是lexical的（静态的），this指向由箭头函数的外围决定
  2. 传统函数的this是动态的，this指向由其调用方式决定

5.4、使用表达式（expression）或statement作为表达式函数体

JS中

3+4
foo(7)
'abc'.length

都是表达式

JS中statement是指类似这样的：

while(true){...}
return 123;

很多表达式都可以写成statement，只要将它们写在statement位置：

function foo(){
    3+4;
    foo(7);
    'abc'.length;
}

5.5、什么时候函数体可以省去花括号？

  1. 如果函数体是表达式则可以省去花括号
  2. 如果函数体是statement则不可以省略

5.7、箭头函数和普通函数的主要区别

箭头函数和普通函数的主要区别体现在两点：

  1. this、super、new.target、arguments是lexical的，这个在前面已经说过
  2. 箭头函数不可以当做构造函数被调用，new (()=>{})将会报错。

除此之外和普通函数没什么差别。 使用typeof 和 instance of得到的结果是一样的。
