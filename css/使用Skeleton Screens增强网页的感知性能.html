使用Skeleton Screens增强网页的感知性能

一、何为网页“感知性能”？

网页的感知性能主要指的是用户在使用网页时感知到的网页加载速度等方面的性能。简而言之就是网页加载时间。

据说MIT对此做过一项研究，结论表明人类可以在13毫秒内感知离散图像，但决定将眼睛聚焦到哪里需要100到140毫秒。 实际上，这为我们提供了大约200毫秒的时间来呈现网页状态。在200毫秒至1s的时间内，用户绝得他们处在这个过程中，超过1s之后他们就会失去耐心了。

当网络性能较差或页面本身问题导致加载时间较长时，为了避免让用户失去耐心，通常的做法是使用一个进度条来使用户处在一个可预期的等待过程中，这种方法可以较好的增强网页的感知性能。

对于可以获取到有效进度值的场景，使用进度条可以起到很好的效果，而对于无法获取有效进度值的场景，我们常常会使用loading状态来处理。但是无有效进度值的loading状态显然不是一种很好的方式。

除了以上这些方法用来提高网页感知性能外，业界还提出了一种叫做“Skeleton Screens”的解决方案。

二、何为“Skeleton Screens”？

所谓Skeleton Screens，其实就是指的网页的基本骨架，包括文本元素区域，图像元素区域以及其他元素区域等等。Skeleton Screens不包含网页的实际元素内容，仅仅包含元素的骨架。

每当我们从服务器请求新内容时，可以立即开始显示网页骨架，而数据正在后台加载。 一旦内容准备就绪，只需将骨架替换为实际的内容即可。 这可以通过简单的JavaScript或者使用像React这样的库来完成。

Skeleton Screens可以改善任何超过几百毫秒渲染的体验。 用它来解决渲染瓶颈可以使我们的用户界面感觉更快，并用户体验更爽。

三、如何实现Skeleton Screens？

实现Skeleton Screens的最简单的方式当然就是使用UI给我切好的图片了，但是呢使用图片其实还是会需要一次额外的请求，并且使用图片不具有扩展性，因为当我们的骨架有变动的时候，还需要重新切图。

另一种较好的方式是，使用纯CSS构建整个骨架，没有额外的网络请求，而且扩展性好，这样我们就可以很好的维护Skeleton Screens了。

从前面的案例我们也可以看到，Skeleton Screens基本上都是一些灰白的元素堆叠起来的，这个很适合用CSS的background属性来实现，因为我们可以设置多个background值。

举个例子，假设我们想实现一个下面这个UI的Skeleton Screens：

它的实现只需要一个元素就可以了：

然后他的CSS可以这样写：

.card {
    height: 340px;
    width: 280px;
    padding: 24px;
    border-radius:6px;
    box-shadow: 0 10px 45px rgba(0,0,0, .1);
    background-repeat: no-repeat;
    background-image: 
        linear-gradient(90deg, rgba(211,211,211, 0) 0, rgba(211,211,211, 0.5) 50%, rgba(211,211,211, 0) 100%),
        /* layer 2: avatar */
        /* white circle with 16px radius */
        radial-gradient(circle 16px, white 99%, transparent 0),
        /* layer 1: title */
        /* white rectangle with 40px height */
        linear-gradient(white 40px, transparent 0),
        linear-gradient(white 20px, transparent 0),
        linear-gradient(white 20px, transparent 0),
        linear-gradient(white 40px, transparent 0),
        /* layer 0: card bg */
        /* gray rectangle that covers whole element */
        /* linear-gradient(gray 100%, transparent 0); */
        linear-gradient( rgba(211,211,211, 0.8) 100%, transparent 0);
        /* linear-gradient( rgba(128, 128, 128, 0) 0, rgba(128, 128, 128, 0.8) 50%, rgba(128, 128, 128, 1) 100%); */

    background-size:
        200px 350px,
        32px 32px,  /* avatar */
        200px 40px, /* title */
        260px 20px, /* title */
        180px 20px, /* title */
        100% 100%, /* title */
        100% 100%;  /* card bg */
    background-position:
        -150% 0,
        24px 24px,  /* avatar */
        24px 200px, /* title */
        24px 270px,
        24px 310px,
        0 350px,
        0 0;        /* card bg */
    animation: loading 1.5s infinite;
}
@keyframes loading {
    to {
        background-position:
        350% 0,       
        24px 24px,  /* avatar */
        24px 200px, /* title */
        24px 270px,
        24px 310px,
        0 350px,
        0 0;        /* card bg */
        ;
    }    
}

注意，为了使我们的骨架看起来更灵动，我们还加了动画，这样可以更好的改善用户体验。

四、使用CSS自定义变量实现Skeleton Screens

上面的代码看起来非常的混乱，维护起来不是特别容易，所以我们可以考虑使用CSS自定义变量将它们进行改进，CSS自定义变量支持大部分的现代浏览器。如果你对它还不了解，你可以参考一下。

我们首先定义CSS变量：

/*
 * Variables
 */

:root {  
  //卡片骨架变量
  --card-padding: 24px;//卡片padding
  --card-height: 340px;//卡片高度
  --card-skeleton: linear-gradient(rgba(211,211,211,1) var(--card-height), transparent 0);//卡片骨架
  //头像骨架变量
  --avatar-size: 32px;
  --avatar-position: var(--card-padding) var(--card-padding);
  --avatar-skeleton: radial-gradient(circle 16px at center, white 99%, transparent 0
  );
  //标题骨架变量
  --title-height: 32px;
  --title-width: 200px;
  --title-position: var(--card-padding) 180px;
  --title-skeleton: linear-gradient(white var(--title-height), transparent 0);
  //描述骨架变量
  --desc-line-height: 16px;
  --desc-line-skeleton: linear-gradient(white var(--desc-line-height), transparent 0);
  --desc-line-1-width:230px;
  --desc-line-1-position: var(--card-padding) 242px;
  --desc-line-2-width:180px;
  --desc-line-2-position: var(--card-padding) 265px;
  //底部栏骨架变量
  --footer-height: 40px;
  --footer-position: 0 calc(var(--card-height) - var(--footer-height));
  --footer-skeleton: linear-gradient(white var(--footer-height), transparent 0);
  //动画骨架变量
  --blur-width: 200px;
  --blur-size: var(--blur-width) calc(var(--card-height) - var(--footer-height));
}

上面只是定义了我们即将用到的变量，接下来我们定义card骨架的实际样式：

/*
 * Card Skeleton for Loading
 */

.card {
  width: 280px; //demo
  height: var(--card-height);
}

对于card上的其它元素，我们采用after的方式定义：

.card::after {
    content:"";
    display:block;
    width: 100%;
    height: 100%;
    border-radius:6px;
    box-shadow: 0 10px 45px rgba(0,0,0, .1);

    background-image:
      linear-gradient(
        90deg, 
        rgba(211,211,211, 0) 0, 
        rgba(211,211,211, .8) 50%, 
        rgba(211,211,211, 0) 100%
      ),                          //animation blur
      var(--title-skeleton),      //title
      var(--desc-line-skeleton),  //desc1
      var(--desc-line-skeleton),  //desc2
      var(--avatar-skeleton),     //avatar
      var(--footer-skeleton),     //footer bar
      var(--card-skeleton)        //card
    ;

    background-size:
      var(--blur-size),
      var(--title-width) var(--title-height),
      var(--desc-line-1-width) var(--desc-line-height),
      var(--desc-line-2-width) var(--desc-line-height),
      var(--avatar-size) var(--avatar-size),
      100% var(--footer-height),
      100% 100%
    ;

    background-position:
      -150% 0,                      //animation
      var(--title-position),        //title
      var(--desc-line-1-position),  //desc1
      var(--desc-line-2-position),  //desc2
      var(--avatar-position),       //avatar
      var(--footer-position),       //footer bar
      0 0                           //card
    ;

    background-repeat: no-repeat;
    animation: loading 1.5s infinite;
}

同样，我们也是需要定义动画：

@keyframes loading {
  to {
    background-position:
      350% 0,        
      var(--title-position),  
      var(--desc-line-1-position),
      var(--desc-line-2-position),
      var(--avatar-position),
      var(--footer-position),
      0 0
    ;
  }
}

五、结语

Skeleton Screens是一种改善网页用户体验的技术，它的作用尤其体现在网络性能较差的时候，很多网站都采用了这种技术，如果你想让你的网页具有更好的感知性能，赶快加入吧！
