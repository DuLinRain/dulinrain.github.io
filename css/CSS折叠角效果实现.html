CSS折叠角效果实现

设计一种类似于纸张的某一个角被折叠的效果是大家会经常会遇到的场景，话不多说先看效果：

最早实现这种效果的应该是Nicolas Gallagher，但是他们的方法大多都是用三个元素来实现：一个主元素和2个三角形。其中一个三角形设置为和主元素一样的颜色来盖住主元素的右上角，另外一个三角形用于模拟折叠效果。 但这种实现方法会有以下问题：

  1. 当主元素不是纯色，而是一个pattern、纹理、图片、渐变色等的时候，你拿什么去盖住主元素的右上角？
  2. 当折叠角不是恰好45度的时候咋办？

那么有没有适应性更好的实现方案呢？当然是有的，我们先从45度角的情况说起。

1. 45度折叠效果实现

我们这里只用一个主元素来实现。 主元素的右上角不再使用三角形来盖住形成裁剪效果，而是使用径向渐变将右上角设置为透明度：

html代码：

<div id="clip-conor"></div>

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade浏览器不支持时降级为纯色效果*/
    background: linear-gradient(-135deg, transparent 50px, #58a 0);
}

最终效果：

现在我们已经有了裁减掉的角，我们现在只需要用一个深色的三角形衔接到右上角即可。 这个三角形也不需要额外的元素，仍然是采用径向渐变的背景来实现的。 我们知道用径向渐变实现三角形只需要在50处设置hardstop color即可，同样是设置它从-135度渐变，指定背景大小为20px 20px(因为是正方形，所以-135度也可以直观地写成 to left bottom)：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0 / 50px 50px no-repeat,
        linear-gradient(-135deg, transparent 50px, #58a 0);
}

效果：

这下结果虽然是朝着我们期望的方向去的，但是，大小好像不匹配。为什么会这样呢？ 这是因为，前面的主元素右上角，我们指定透明渐变的长度为50px，实际上这个50px的意思是沿着-135度渐变线的长度, 结合基本的三角公式，我们知道实际上水平/垂直方向的长度应该乘以根号2(即1.414)，而我们在第二个径向渐变的时候指定的background-size的意义就是指水平方向和垂直方向，所以这个问题的解决办法自然有两种：

  1. 第二个径向渐变的background-size保持不变，第一个的长度除以根号2.
  2. 第二个径向渐变的background-size乘以根号2，第一个的长度不变.

两种都可以，但是考虑到background-size属性是两个值，这样就得改两次，我们就采用第一种方法得到的长度是约等于35px。这样得到的效果就出来了：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0 / 50px 50px no-repeat,
        linear-gradient(-135deg, transparent 35px, #58a 0);
}

效果：

2. 对于其它角度呢？

事实上，45度角与实际的折叠场景对比并不是很真实，实际我们可能是各种角度，所以假如角度变为-150度会怎样呢？我们直接改变角度值试一试：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0 / 50px 50px no-repeat,
        linear-gradient(-150deg, transparent 35px, #58a 0);
}

效果：

可以看到， 大小又不匹配了，同样我们需要如下三角公式进行计算：

我们只需要将三角形的background-size改变成计算的值即可：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0 / 70px 40px no-repeat,
        linear-gradient(-150deg, transparent 35px, #58a 0);
}

效果：

但是这个折叠效果看起来很不真实，不信你可以自己拿张值来折叠一下就会发现问题所在了。比如真实的应该像下面这样，有个旋转角度的：

但是我们又没法旋转background-image，所以前面那种用两个background-image的方法是行不通了的。 什么东西可以满足既可以前面的效果，又可以旋转呢？是时候让伪元素出马了。 现在我们需要做的是将右上角的三角形换成用伪元素来表示，这个伪元素绝对定位于右上角，任然采用径向渐变生成三角形效果，但是伪元素的宽高和前面背景的宽高正好调换顺序：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    position: relative;/*作为伪元素绝对定位的包含块*/
    background: linear-gradient(-150deg, transparent 35px, #58a 0);
}
#clip-conor::before {
    content: '';
    position: absolute;
    top: 0;
    right: 0;
    width: 40px;
    height: 70px;
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0  no-repeat;
}

效果：

貌似我们只需要旋转30度就OK了， 我们来试试：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    position: relative;/*作为伪元素绝对定位的包含块*/
    background: linear-gradient(-150deg, transparent 35px, #58a 0);
}
#clip-conor::before {
    content: '';
    position: absolute;
    top: 0;
    right: 0;
    width: 40px;
    height: 70px;
    transform: rotate(-30deg);
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0  no-repeat;
}

效果：

结果并不是我们想要的，我们大概可以看出来，这个需要水平、垂直方向都进行移动，但是移动多少呢？这个样子就很难进行计算了。这是因为transform: rotate(-30deg)的时候，是沿着中心进行的，我们将它设置为沿着右下角旋转就容易计算多了：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    position: relative;/*作为伪元素绝对定位的包含块*/
    background: linear-gradient(-150deg, transparent 35px, #58a 0);
}
#clip-conor::before {
    content: '';
    position: absolute;
    top: 0;
    right: 0;
    width: 40px;
    height: 70px;
    transform: rotate(-30deg);
    transform-origin: bottom right;
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0  no-repeat;
}

效果：

可以看到，右下角是没有动的，现在我们只需要沿着垂直方向移动就可以了，移动多少呢？ 我们可以用旋转前的图形效果进行计算：

从这个示意图我们可以看出来，只需要先向上移动高度-宽度这么多就可以了。对于我们的例子也就是30px。 我们尝试一下：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    position: relative;/*作为伪元素绝对定位的包含块*/
    background: linear-gradient(-150deg, transparent 35px, #58a 0);
}
#clip-conor::before {
    content: '';
    position: absolute;
    top: 0;
    right: 0;
    width: 40px;
    height: 70px;
    transform: translateY(-30px) rotate(-30deg);
    transform-origin: bottom right;
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.4) 0) 100% 0  no-repeat;
}

效果：

这里要注意的是，translateY要在rotate前面，另外，舍入误差也会对连接处造成影响。

最后，我们可以给他加border-radius、box-shadow或者改变渐变色来看起来更好看：

css代码：

#clip-conor{
    width: 300px;
    height: 200px;
    background: #58a; /*degrade*/
    position: relative;/*作为伪元素绝对定位的包含块*/
    background: linear-gradient(-150deg, transparent 35px, #58a 0);
    border-radius: 10px;
}
#clip-conor::before {
    content: '';
    position: absolute;
    top: 0;
    right: 0;
    width: 40px;
    height: 70px;
    transform: translateY(-30px) rotate(-30deg);
    transform-origin: bottom right;
    background: linear-gradient(to left bottom, transparent 50%, rgba(0,0,0,.2) 0, rgba(0,0,0,.4)) 100% 0  no-repeat;
    border-bottom-left-radius: inherit;
    box-shadow: -5px 6px 5px -3px rgba(0,0,0,0.15);
}

结果：

但是以上代码其实看起来并不具有很好的适应性，所以为了尽可能少针对不同的场景做修改，最好还是使用预处理语言，因为他们可以用很多变量以及执行基本的加减乘除计算。
