CSS变量入门

一、概述

CSS变量和任何其他编程语言中的变量一样，使得我们可以在CSS中的其它地方对他们进行引用。

这听起来很不可思议，你也许会觉得，这不是需要借助于SASS、LESS之类的预处理语言才能实现的吗？

幸运的是，从2017年4月开始，所有的现代浏览器都开始支持CSS变量了。使用CSS变量可以使我们的CSS代码变得更加干净和紧凑。

你可能已经比较想要知道我们怎么样才能使用CSS变量了，下面就跟随本文一起来了解一下吧！

二、如何使用CSS变量

2.1、 定义CSS变量

任何CSS属性-color、size、position等等-都可以用CSS变量来保存。他们的属性名字前面有一个“--”前缀，其他和普通属性没什么区别。 比如：


body {
  --primary: #7F583F;
  --secondary: #F7EFD2;
}

上面的代码，我们定义了两个变量：--primary，--secondary。

2.2、 使用CSS变量

那么我们如何在CSS中的其它的地方引用它们呢？我们需要像下面这样用var()包裹它即可：


a {
  color: var(--primary);
  text-decoration-color: var(--secondary);
}

如果你在你的浏览器的调试窗口中查看这个CSS，你会发现浏览器没有对这些变量做任何的转换。因为浏览器自己能够识别CSS变量。

2.3、 给CSS变量提供默认值

我们在使用CSS变量的时候，也可以给CSS变量提供默认值，只需要在var()的第二个参数中传入即可：


color: var(--primary, #7F583F);

那么默认值会在什么情况下被采纳呢？有两种情况：

  1. 变量未定义
  2. 变量在当前作用域中不存在

这里提到了作用域的概念，我们接下来就来看看什么是作用域吧。

三、作用域和层叠

CSS变量的作用域和普通属性的作用域一样，都是在当前层及其所有孩子层有效。比如说我们定义在body元素下的这个变量，我们在整个页面都能访问到：


body {
  --primary: #7F583F;
  --secondary: #F7EFD2;
}

并且如果变量定义在class里面的话，这些变量只能在拥有这个class的元素上使用：


.content {
  --primary: #7F583F;
  --secondary: #F7EFD2;
}

如果我们像上面这么在class里定义CSS变量的话，那么我们只能在拥有该class的元素上使用，否则相应的样式不会被渲染。

四、术语：自定义CSS属性

到目前为止，你已经看到了一种场景——CSS变量和CSS属性表现的几乎一样。我们就是在设置属性的地方声明的变量，它们也有着一样的层叠规则。

这是因为CSS变量实际上就是CSS自定义属性。它和普通属性的唯一区别就在于普通属性的值总是固定的，而CSS自定义属性的值是在使用时才确定的。

浏览器支持CSS变量意味着我们可以设置任意的、带命名空间的属性。这是非常有意思的，就好比我们可以使用media query绕开使用js监听resize事件，CSS给我们打开了一扇脱离less等预处理语言的大门。

五、比SASS更好 ——【在线主题切换】

CSS变量和SASS变量不完全同，而且在某些方面，前者比后者更好。其中一个场景是【在线主题切换】。

在线主题切换使用SASS非常容易实现：把你的颜色组合存储在SASS maps中，然后遍历，选择你想要的主题应用到页面中：


.theme-1 {
  a {
    color: #7F583F;
    text-decoration-color: #F7EFD2;
  }
}

.theme-2 {
  a {
    color: #D51522;
    text-decoration-color: #F4F6D8;
  }
}

/* etc */

但是，这会导致我们使用其中一个主题的时候，剩余的主题的代码实际上都是无用的。

CSS变量可以实现同样的效果，并且保证没有额外的代码。 我们不用通过js给页面添加主题class，而是通过JS直接改变变量的值：


document.body.style.setProperty('--primary', '#7F583F');
document.body.style.setProperty('--secondary', '#F7EFD2');

这些改变会应用到所有使用了这些变量的元素。你的CSS代码保持的很干净，而且我们是直接改变的CSS，没有中间商赚差价😁。

六、比SASS更好 ——【meida query】

在SASS中，在media query中重新定义变量是不允许的。 比如你想在某种条件下重新定义link的颜色，在SASS中你可能会这么写代码：


$primary: #7F583F;
$secondary: #F7EFD2;

a {
  color: $primary;
  text-decoration-color: $secondary;

  @media screen and (min-width: 768px) {
    $primary: #F7EFD2;
    $secondary: #7F583F;
  }
}

然后你会尴尬的发现，这个在SASS中不会生效，这是因为SASS是预处理语言，也就是预先处理，它没法知道未来会发生什么情况。

但是， 我们可以使用CSS变量实现这种效果：


body {
  --primary: #7F583F;
  --secondary: #F7EFD2;
}

a {
  color: var(--primary);
  text-decoration-color: var(--secondary);
}

@media screen and (min-width: 768px) {
  body {
    --primary:  #F7EFD2;
    --secondary: #7F583F;
  }
}

这可以生效的原因是，所有的工作都是浏览器帮我们做的，因而这些变量也知道他们在何种条件下在使用。

七、浏览器支持

CSS变量最初在2014年Firefox浏览器中得到支持，2016年在Chrome和Safari中得到支持。所以好消息是他们的支持很广泛，坏消息仍然是：IE还是个令人头疼的孩子，几乎是所有的IE。

八、提供fallback

幸运的是，给他们提供fallback的方式和我们在远古时代使用的方式一样：


a {
  color: #7F583F;
  color: var(--primary);
}

先声明你的fallback，然后声明自定义变量。

九、使用SASS更容易实现fallback

如果你使用SASS，你可以通过mixin自动实现fallback。创建你的自定义变量名和值的映射，然后你就可以动态的去查找它们并且输出最终的样式了：


$vars: (
  primary: #7F583F,
);

body {
  --primary: #{map-get($vars, primary)};
}

@mixin var($property, $varName) {
  #{$property}: map-get($vars, $varName);
  #{$property}: var(--#{$varName}, map-get($vars, $varName));
}

上面的mixin可以像下面这么使用：


a {
  @include var(color, primary);
}

并且输出如下CSS：


a {
  color: #7F583F;
  color: var(--primary, #7F583F);
}

如果你想改变--primary或者是它的fallback,你只需要改变$vars即可。
