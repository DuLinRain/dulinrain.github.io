ElementUI中clickoutside.js的实现

1、 概述

我们在写前端的时候经常遇到这样一种场景，希望鼠标点击到元素外面的时候执行某种操作，比如说，点击空白处，菜单收起。这种场合其实是很多的，ElementUI将这种场景进行了抽象，巧妙地运用Vue的自定义指令的方式实现了这样的功能，该指定的使用方式如下：

/**
 * v-clickoutside
 * @desc 点击元素外面才会触发的事件
 * @example
 * ```vue
 * <div v-element-clickoutside="handleClose">
 * ```
 */

2、重温Vue自定义指令

要想先看源码得先知道vue中自定义指定是如何使用的，其实不用指定我们也可以完成类似的功能，但是指令使得它实现起来更通用，更方便。下面我们复习一下vue官网对自定义指令的描述。

除了核心功能默认内置的指令 (v-model 和 v-show)，Vue 也允许注册自定义指令。注意，在 Vue2.0 中，代码复用和抽象的主要形式是组件。然而，有的情况下，你仍然需要对普通 DOM 元素进行底层操作，这时候就会用到自定义指令

指令的注册方法非常简单，全局注册的话直接使用Vue.directive(name, options)的形式：

// 注册一个全局自定义指令 `v-focus`
Vue.directive('focus', {
  // 当被绑定的元素插入到 DOM 中时……
  inserted: function (el) {
    // 聚焦元素
    el.focus()
  }
})

局部注册的话只需要在组件中定义directives属性，类似如下：

directives: {
  focus: {
    // 指令的定义
    inserted: function (el) {
      el.focus()
    }
  }
}

注册完后只需要像下面这样使用就可以了：

<input v-focus type="text" value="">

指令也可以接收参数和修饰符，比如v-testdrective:foo.a.b.x中，foo就是参数，a, b, x 就是修饰符。

3、ElementUI中v-clickoutside的实现

ElementUI中v-clickoutside是在clickoutside.js中实现的，我们来看看它的源码：

1 ~ 14行源码如下：

import Vue from 'vue';
import { on } from 'element-ui/src/utils/dom';

const nodeList = [];
const ctx = '@@clickoutsideContext';

let startClick;

!Vue.prototype.$isServer && on(document, 'mousedown', e => (startClick = e));

!Vue.prototype.$isServer && on(document, 'mouseup', e => {
  nodeList.forEach(node => node[ctx].documentHandler(e, startClick));
});

该源码引入了我们在ElementUI中Dom.js源码阅读中定义的事件绑定函数on。 上述这段代码先在document上注册了mousedown和mouseup两个事件的事件响应函数。 定义了上下文常量ctx，用startClick保存mousedown事件，定义了nodeList存储触发事件的dom，也就钩子的第一个参数。

后面的代码，就是v-clickoutside自定义指令的真正实现代码，它只使用了三个钩子bind,update,unbind：

/**
 * v-clickoutside
 * @desc 点击元素外面才会触发的事件
 * @example
 * ```vue
 * <div v-element-clickoutside="handleClose">
 * ```
 */
export default {
  bind(el, binding, vnode) {
    const id = nodeList.push(el) - 1;
    const documentHandler = function(mouseup = {}, mousedown = {}) {
      if (!vnode.context ||
        !mouseup.target ||
        !mousedown.target ||
        el.contains(mouseup.target) ||
        el.contains(mousedown.target) ||
        el === mouseup.target ||
        (vnode.context.popperElm &&
        (vnode.context.popperElm.contains(mouseup.target) ||
        vnode.context.popperElm.contains(mousedown.target)))) return;

      if (binding.expression &&
        el[ctx].methodName &&
        vnode.context[el[ctx].methodName]) {
        vnode.context[el[ctx].methodName]();
      } else {
        el[ctx].bindingFn && el[ctx].bindingFn();
      }
    };
    el[ctx] = {
      id,
      documentHandler,
      methodName: binding.expression,
      bindingFn: binding.value
    };
  },

  update(el, binding) {
    el[ctx].methodName = binding.expression;
    el[ctx].bindingFn = binding.value;
  },

  unbind(el) {
    let len = nodeList.length;

    for (let i = 0; i < len; i++) {
      if (nodeList[i][ctx].id === el[ctx].id) {
        nodeList.splice(i, 1);
        break;
      }
    }
  }
};

bind方法在指令所绑定的dom上定义了ctx属性，在这个属性上挂载了一个对象：

{
  id,
  documentHandler,
  methodName: binding.expression,
  bindingFn: binding.value
}

这个对象包含这个我们给这个dom的编号，这个编号就是按进入nodeList数组的顺序从0开始进行的编号，以及自定义的一个documentHandler函数，以及我们自定义指令所绑定的表达式的名字，对于下面的例子而言, 表达式的名字就是指handleClose。以及表达式的值, 对于下面这个例子而言，表达式的值就是指我们在组件中实际定义的handleClose方法。

/**
 * v-clickoutside
 * @desc 点击元素外面才会触发的事件
 * @example
 * ```vue
 * <div v-element-clickoutside="handleClose">
 * ```
 */

回过头来继续看documentHandler函数，这个函数首先判断用户点击的是不是el自身，如果是的话直接返回，不做任何处理。如果点击的不是所绑定的元素本身则，则分两种情况，如果vnode.context中已经有了我们所绑定的方法，则执行vnode.context上的相应方法，如果没有则直接执行我们绑定的bindingFn。

接下来看update钩子的实现，update钩子主要用于更新我们前面给dom，也就是el，绑定的那个对象，主要就是methodName和bindingFn属性：

  update(el, binding) {
    el[ctx].methodName = binding.expression;
    el[ctx].bindingFn = binding.value;
  },

最后使用到的一个钩子就是unbind钩子，用于将对应dom所绑定的事件从nodeList中移除：

unbind(el) {
    let len = nodeList.length;

    for (let i = 0; i < len; i++) {
      if (nodeList[i][ctx].id === el[ctx].id) {
        nodeList.splice(i, 1);
        break;
      }
    }
}
