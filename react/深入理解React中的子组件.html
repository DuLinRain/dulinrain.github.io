深入理解React中的子组件

概述

我们在Function-as-Child组件这一篇文章中已经看到了可以使用一个函数作为React子组件以及这样做的一些优势。 凑巧在学习React-redux的Provider组件的时候，我也恰好发现了下面这么一段代码：

render() {
  return Children.only(this.props.children)
}

所以，借此机会学习一下React中的子组件。

一、 子组件

假设我们有一个<Grid />组件，它包含<Row />组件，你可能会像下面这样使用它：

<Grid>
  <Row />
  <Row />
  <Row />
</Grid>

最终结果可能长这样：

其中，<Row />组件长下面这个样子：

class Row extends React.Component {
  render() {
    return <div className="row">Row</div>
  }
}

实际上，上面的三个<Row />组件是以props.children的形式传给<Grid />组件的。<Grid />组件可以通过一个表达式渲染出这几个子组件：

class Grid extends React.Component {
  render() {
    return <div className="grid">{this.props.children}</div>
  }
}

当然， <Grid />组件也可以完全忽略这些子组件，不渲染任何一个：

class Grid extends React.Component {
  render() {
    return <div className="grid">no child is render</div>
  }
}

二、 任何东西都可以成为子组件

子组件不一定非得是React组件，任何东西都可以成为子组件, 比如说，可以把字符串作为子组件：

<Grid>
    Hello World
</Grid>


class Grid extends React.Component {
  render() {
    return <div className="grid">{this.props.children}</div>
  }
}

JSX会自动的删除空格，行起始、结束符，以及空白行，也就是说，下面的几种写法是等价的：

<Grid>Hello world!</Grid>

<Grid>
  Hello world!
</Grid>

<Grid>
  Hello
  world!
</Grid>

<Grid>

  Hello world!
</Grid>

子组件并不是每次只能有一种，也可以多种混合，比如：

<Grid>
  Here is a row:
  <Row />
  Here is another row:
  <Row />
</Grid>

函数作为子组件

函数作为子组件也就是我们在Function-as-Child组件这一篇文章中所介绍的。 这里再简单说一下。 在React中，你可以把任何JS表达式作为子组件，这其中自然包括函数。比如下面这个例子：

class Executioner extends React.Component {
  render() {
    // See how we're calling the child as a function?
    //                        ↓
    return this.props.children()
  }
}

<Executioner/>组件会把this.props.children作为函数执行的结果渲染出来，也就是说它期待子组件是一个函数。 这意味着，我们可以像下面使用该组件：

<Executioner>
  {() => <h1>Hello World!</h1>}
</Executioner>

这个组件没有任何实际的意义，但是它明确无误的表达了这种思路。 结合Function-as-Child组件这篇文章你会发现函数作为子组件的意义。

四、 操作子组件

实际上上面所讲的都可能是大家早就熟知的东西，下面将会讲到React中用来操作子组件的一些API。 这其中也包括我们在开篇提到的Redux的Provider组件中用到的那个API。

如果你看一看React的文档，你会发现对子组件的描述是“他们是透明的数据结构”，它实际上想要表达的是： props.children可以是任意的数据类型，如Array,Object，Function等等。 由于你可以传入任何东西，所以这是一个不确定的东西。 也正因为此，Rreact提供了一系列的辅助方法用于操作子组件，这些方法都挂载在React.Children上。

4.1、 React.Children.map和React.Children.forEach遍历子组件

React提供了React.Children.map和React.Children.forEach2个辅助方法用来遍历子组件。他俩的用法和原生Array的map和forEach基本一样，区别在于，当传过来的子组件是Function，Object等类型的时候它们仍然能够正常工作。

遍历子组件的一个用处在于我们可以操作子组件的各种行为，比如说，下面这个组件不会渲染前2个子组件：

class Grid extends React.Component {
  render() {
    let children  = this.props.children;
    return <div className="grid">
        {React.Children.map(children, (child, i) => {
          // Ignore the first child
          if (i < 2) return
          return child
        })}</div>
  }
}

<Grid>
    first Row
    <Row />
    second Row
    <Row/>
</Grid>

实际上，上面这种情况我们也可以直接使用this.props.children.map来遍历：

class Grid extends React.Component {
  render() {
    let children  = this.props.children;
    return <div className="grid">
        {children.map((child, i) => {
          // Ignore the first child
          if (i < 2) return
          return child
        })}</div>
  }
}

但是，如果我们的子组件是函数的话会怎样呢？

<Grid>
    {() => (<Row />)}
</Grid>

我们会发现控制台报错了：

但是如果我们使用React.Children.map的话，至少是不会报错的。

4.2、 React.Children.count计算子组件的个数

当遇到计算子组件的个数的时候，很多人第一想到的就是props.children.length，但是当子组件是字符串或函数的时候，这样子很定会计算错误的。 为此，React提供了React.Children.count专门用于计算子组件的个数：

<Grid>
string Component
{()=> <h1>Function Component</h1>}
<Row/>
</Grid>


class Grid extends React.Component {
  render() {
    var children  = this.props.children;
    return <div className="grid">{ React.Children.count(children)}</div>
  }
}

注意： 不知道为啥，我实际测试，发现不会计算函数组件。

4.3、 React.Children.toArray将子组件转为数组操作

React.Children.toArray方法使你可以将子组件转换成数组的元素，这样你可以做类似如下这样排序子组件&拼接的操作：

class Grid extends React.Component {
  render() {
    const children = React.Children.toArray(this.props.children)
    return <div className="grid">{ children.sort().join(' ')}</div>
  }
}

<Grid>
{'C'}
{'B'}
{'A'}
</Grid>

4.4、 React.Children.only强制单个子组件

React.Children.only可以强制只接受一个子组件，如果传递了多个子组件会报错：

class Grid extends React.Component {
  render() {
    return <div className="grid">{ React.Children.only(this.props.children)}</div>
  }
}

<Grid>
{'C'}
{'B'}
{'A'}
</Grid>

看到这里你就能理解开篇Redux中Provider组件那种写法的含义了。

4.5 React.cloneElement克隆并操作子组件的属性

假设我们要做一个RadioGroup组件，你可能会像下面这样子写：

class RadioGroup extends React.Component {
  render() {
    return <div className="grid">{ this.props.children}</div>
  }
}
class RadioButton extends React.Component {
  render() {
    return <React.Fragment>
            <input type="radio" value={this.props.value}/>
            {this.props.children}
           </React.Fragment>
  }
}
ReactDOM.render(
    <RadioGroup>
        <RadioButton value="first">first</RadioButton>
        <RadioButton value="second">second</RadioButton>
        <RadioButton value="last">second</RadioButton>
    </RadioGroup>,
    document.getElementById('example')
);

但是，你会发现他们是彼此独立的，并不是Group：

你可能会像下面这么做改进：

class RadioButton extends React.Component {
  render() {
    return <React.Fragment>
            <input type="radio" value={this.props.value} name={this.props.name}/>
            {this.props.children}
           </React.Fragment>
  }
}
ReactDOM.render(
    <RadioGroup>
        <RadioButton name="g1" value="first">first</RadioButton>
        <RadioButton name="g1"value="second">second</RadioButton>
        <RadioButton name="g1"value="last">second</RadioButton>
    </RadioGroup>,
    document.getElementById('example')
);

但更好的办法是统一在RadioGroup上定义name属性，然后只要我们将子组件的name设置成一样就行了，我们可以写成这样：

class RadioGroup extends React.Component {
  renderChildren() {
    return React.Children.map(this.props.children, child => {
        // TODO: Change the name prop to this.props.name
        return child
    })
  }
  render() {
    return <div className="grid">{ this.props.children}</div>
  }
}
class RadioButton extends React.Component {
  render() {
    return <React.Fragment>
            <input type="radio" value={this.props.value} name={this.props.name}/>
            {this.props.children}
           </React.Fragment>
  }
}
ReactDOM.render(
    <RadioGroup name="g1">
        <RadioButton value="first">first</RadioButton>
        <RadioButton value="second">second</RadioButton>
        <RadioButton value="last">second</RadioButton>
    </RadioGroup>,
    document.getElementById('example')
);

现在我们唯一还差的就是下面这个函数：

renderChildren() {
    return React.Children.map(this.props.children, child => {
        // TODO: Change the name prop to this.props.name
        return child
    })
}

我们如何给子组件定义一个name属性呢？ 这就是React.cloneElement派上用场的时候了，React.cloneElement可以克隆并操作子组件的属性，所以我们可以写成这样：

renderChildren() {
    return React.Children.map(this.props.children, child => {
        return React.cloneElement(child, {
        name: this.props.name
        })
    })
}

最终的代码如下：

class RadioGroup extends React.Component {
  renderChildren() {
    return React.Children.map(this.props.children, child => {
        return React.cloneElement(child, {
            name: this.props.name
        })
    })
  }
  render() {
    return <div className="grid">{ this.props.children}</div>
  }
}
class RadioButton extends React.Component {
  render() {
    return <React.Fragment>
            <input type="radio" value={this.props.value} name={this.props.name}/>
            {this.props.children}
           </React.Fragment>
  }
}
ReactDOM.render(
    <RadioGroup name="g1">
        <RadioButton value="first">first</RadioButton>
        <RadioButton value="second">second</RadioButton>
        <RadioButton value="last">second</RadioButton>
    </RadioGroup>,
    document.getElementById('example')
);

我们达到了同样的效果。
