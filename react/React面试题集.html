React面试题集

1、 什么是React?

React是fb开源的JS库，用于构建复杂的web用户界面。它的核心目的是构建UI组件，通常对应于MVC模式中的V(View)。

2、 什么是Redux？

Redux是为js应用提供可预测的状态管理器，通常认为它受Flux启发。

3、 什么是虚拟DOM？

虚拟DOM（VDOM）是真实DOM在内存中的表示。 UI的展示保存在内存中并与真实DOM同步。 它发生在调用render函数与将元素显示在屏幕 这两个步骤之间。 整个过程也称为reconciliation。

4、 什么是Flux?

Flux是一种应用程序设计范例，用于替代更传统的mvc模式。 它不是一个框架或库，而是一种新的体系结构，它提出了单向数据流的概念。 在使用React时，Facebook在内部使用此模式调度程序，存储和视图组件之间的工作流程具有不同的输入和输出，如下所示：

5、 React的优势是什么？

React有以下几个优势：

  1. 通过Virtual DOM提升了应用程序的性能
  2. JSX语法使得代码更好读写
  3. 既可以在客户端render也可以在服务端render，这位ssr提供了可能。

6、 React的主要特性？

  1. 使用虚拟DOM代替真实DOM。
  2. 支持SSR
  3. 遵循单向数据流或数据绑定
  4. 使用可重用组件构建复杂UI

7、 表示组件和容器组件有什么区别?

  • 表示组件-顾名思义，表示组件只关心UI长什么样(how things look)，他们通常只通过props接收数据和callback。 这些组件很少拥有自己的state，即使有也仅仅用于表示UI状态而不是数据状态。
  • 容器组件-容器组件更关心如何工作(how things work)。 容器组件为表示组件或其它容器组件提供数据与行为。它们通常也是有状态的，因为它们充当数据源。

8、 描述一下React是如何处理事件的？

为了解决跨浏览器兼容性问题，React中的事件处理程序将传递合成事件的实例，这是React围绕浏览器原生事件的跨浏览器封装。这些合成事件与我们习惯的原生事件具有相同的接口，并且它们在所有浏览器中的工作方式相同。

更重要的是，React并不是为每个组件/元素绑定对应的事件，而是采用流行的事件委托方式，使用单个事件侦听器绑定在根元素侦听顶级的所有事件。 这对性能有好处，这也意味着React不需要担心在更新DOM时跟踪事件监听器。

为每个DOM元素绑定事件是一个非常消耗性能和内存的事情，React也采用了流行的事件委托机制，但是React走的更远。 React在文档的根元素上绑定了事件监听器，当事件触发的时候，浏览器会为我们提供目标DOM的节点，那React是如何将事件传递下去的呢？是在虚拟DOM层级上进行迭代传递的吗？

答案是否定的。

React并不会在虚拟DOM层级上进行迭代，React的每一个层级的组件都有一个编码的ID用于描述它所处的层级，可以通过简单的字符串操作来找到所有的父元素ID，以此可以将事件监听器存储在hash map中，相比于将事件绑定在虚拟DOM上具有更好的性能。下面是一个简单的例子：

// dispatchEvent('click', 'a.b.c', event) //fire一个事件
 clickCaptureListeners['a'](event); //捕获
 clickCaptureListeners['a.b'](event);  //捕获
 clickCaptureListeners['a.b.c'](event); //捕获
 clickBubbleListeners['a.b.c'](event); //冒泡
 clickBubbleListeners['a.b'](event); //冒泡
 clickBubbleListeners['a'](event); //冒泡

浏览器会为每个事件和每个侦听器创建一个新的事件对象。，你可以保持事件对象的引用，甚至修改它。但是，这意味着要进行大量的内存分配。 React在启动的时候分配了一个事件对象池，任何时候需要的时候，都可以进行重用，这极大的减少了垃圾回收。

9、 React中的state是什么？

组件的状态是一个对象，它包含一些可能在组件的生命周期内发生变化的信息。 我们应该总是尽量使我们的状态尽可能简单，并尽量减少有状态组件的数量。

10、 state和props的区别?

props和state都是纯JavaScript对象。 虽然它们都包含影响渲染输出的信息，但它们在组件方面的功能却不同。即：

  • props传递给组件类似于给函数传递参数
  • state之于组件类似于函数中定义的变量

11、 React有什么不足或限制？

  • React只是一个视图框架，而不是一个完整的MVC框架
  • 对于不熟悉Web开发的初学者来说，有一个学习曲线。
  • 将React.js集成到传统的MVC框架中需要一些额外的配置
  • 代码复杂性随内联模板和JSX而增加。
  • 太多较小的组件导致过度工程或样板

12、 在React中元素和组件的区别？

简而言之，React元素描述了您希望在屏幕上看到的内容。 不那么简单地说，React元素是某些UI的对象表示。

React组件是一个函数或类，它可选地接受输入并返回一个React元素（通常通过JSX将其转换为createElement调用）。

18、 JSX是什么？

当Facebook首次向全世界发布React时，他们还引入了一种名为JSX的新方法，它将原始HTML模板嵌入到JavaScript代码中。 浏览器无法读取JSX代码本身; 它必须使用像Babel和webpack这样的工具转换成传统的JavaScript。 虽然许多开发人员可以理解地反对它，但JSX（与ES2015一起）已经成为定义React组件的事实方法。

19、 为什么不能直接修改state？

如果您尝试直接更新状态，不会重新渲染组件。
应该使用setState（）方法。 它会安排对组件状态对象的更新。 当状态改变时，组件通过重新呈现来响应

20、 React组件的生命周期是什么？

React组件的生命周期可分为四大部分：

  1. 实例化。在这个时期组件会设置初始state和默认props
  2. 挂载。组件已经准备好挂载到浏览器DOM中。这一时期包括componentWillMount 和 componentDidMount 2个生命周期方法
  3. 更新。在这一周期，组件通过2种方式实现更新：接收props以及更新state。这一阶段会涉及componentWillReceiveProps,shouldComponentUpdate, componentWillUpdate 和 componentDidUpdate 四个方法。
  4. 卸载。 在这一阶段，组件已经不再需要，被从DOM中移除。这一时期涉及的方法有componentWillUnmount。

21、 React在什么情况下要使用forceUpdate？

如果存在某些条件，React未检测到需要更新UI, 可以调用则forceUpdate强制重新render。 但通常这不需要调用。

24、 React中副作用是什么意思？举几个例子？

“副作用”是影响正在执行的函数范围之外的任何事物。例如，这些可以是网络请求，它使您的代码与第三方通信（从而发出请求，导致记录日志，保存或更新缓存，功能之外的各种效果）。

还有更微妙的副作用。更改闭包范围变量的值是副作用。将新元素push到作为参数传入的数组是副作用。没有副作用的函数称为“纯”函数：它们接受参数，并返回值。执行该功能时没有其他任何事情发生。这使得易于测试，易于推理，并且在优化或重构方面，满足此描述的函数具有各种有用的属性。

纯函数是确定性的（意味着，给定输入，它们总是返回相同的输出），但这并不意味着所有不纯函数都有副作用。例如，在函数内生成随机值会使其不纯，但不会产生副作用。 React完全是关于纯函数的，并要求您确保几个生命周期方法为纯函数。
