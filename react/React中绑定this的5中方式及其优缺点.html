React中绑定this的5中方式及其优缺点

一、概述

React中有很多种绑定this的方法，下面我们就来一一罗列一下。

二、React.createClass中的this

React.createClass中的this会自动的绑定到组件上，因此我们可以直接这么写：

onChange={this.handleChange}

但是随着ES6的语法的到来，渐渐的大家都不再使用React.createClass了。

下面要说的几种方法都假定你使用的是ES6语法定义组件。

三、在render()中使用bind绑定

在使用ES6语法定义的组件时，render中的this不会自动绑定，要想实现绑定，我们可以这样写：

onChange={this.handleChange.bind(this)}

这种方法看起来一目了然，但是从理论上讲，这种方式会影响性能，因为每次render都会进行函数绑定。但是实际上这样做对应用整体的性能影响微乎其微。

如果你真的很在意些许的性能差异，建议你还是不要用这种方法。

四、在render()使用箭头函数绑定

这种方式的写法如下：

onChange={e => this.handleChange(e)}

这样可以实现的原理是箭头函数的词法性，具体我们在这里就不赘述了，想具体了解的可以看这篇文章ES6之箭头函数。

但是，这种方法与上面那种方法存在同样的性能问题。所以如果你特别在意性能的话，可以考虑下面讲的方法。

五、在Constructor中绑定

用法如下：

constructor(props) {
  super(props);
  this.handleChange = this.handleChange.bind(this);
}

这种绑定方法也是React文档中曾经推荐的一种绑定方式，但是其实写起来也是很麻烦，每次感觉都重复写了很多遍的样子。

六、在Class定义属性的时候使用箭头函数绑定

写法如下：

handleChange = () => {
  // call this function from render 
  // and this.whatever in here works fine.
};

这种写法并不是原生支持的，需要使用Babel，你可以安装transform-class-properties插件或者直接安装babel-preset-stage-2预设(preset)。

这种写法有很多优点：

1、箭头函数的this取自外围非箭头函数，所以这种方式实现了自动绑定。
2、它避免了第二种和第三种方法的缺点
3、它避免了第四种方法重复多次的缺点
4、假如你的代码是用React.createClass的形式写的，那么很方便的迁移过来。

七、总结

归结起来可以用下面抄的图描述：

下面是所有5种写法的实际例子：

// Approach 1: Use React.createClass
var HelloWorld = React.createClass({
  getInitialState() {
    return { message: 'Hi' };
  },

  logMessage() {
    // this magically works because React.createClass autobinds.
    console.log(this.state.message);
  },

  render() {
    return (
      <input type="button" value="Log" onClick={this.logMessage} />
    );
  }
});

// Approach 2: Bind in Render
class HelloWorld extends React.Component {
  constructor(props) {
    super(props);
    this.state = { message: 'Hi' };
  }

  logMessage() {
    // This works because of the bind in render below.
    console.log(this.state.message);
  }

  render() {
    return (
      <input type="button" value="Log" onClick={this.logMessage.bind(this)} />
    );
  }
}

// Approach 3: Use Arrow Function in Render
class HelloWorld extends React.Component {
  constructor(props) {
    super(props);
    this.state = { message: 'Hi' };
  }

  logMessage() {
    // This works because of the arrow function in render below.
    console.log(this.state.message);
  }

  render() {
    return (
      <input type="button" value="Log" onClick={() => this.logMessage()} />
    );
  }
}

// Approach 4: Bind in Constructor
class HelloWorld extends React.Component {
  constructor(props) {
    super(props);
    this.state = { message: 'Hi' };
    this.logMessage = this.logMessage.bind(this);
  }

  logMessage() {
    // This works because of the bind in the constructor above.
    console.log(this.state.message);
  }

  render() {
    return (
      <input type="button" value="Log" onClick={this.logMessage} />
    );
  }
}

// Approach 5: Arrow Function in Class Property
class HelloWorld extends React.Component {
  // Note that state is a property,
  // so no constructor is needed in this case.
  state = {
    message: 'Hi'
  };

  logMessage = () => {
    // This works because arrow funcs adopt the this binding of the enclosing scope.
    console.log(this.state.message);
  };

  render() {
    return (
      <input type="button" value="Log" onClick={this.logMessage} />
    );
  }
}
