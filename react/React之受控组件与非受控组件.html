二、HTML原生的form元素行为可视为React中的非受控

原生form表单元素的行为我们都很熟悉了，归结起来就是元素的value由DOM自身处理，无需用户关心，用户想要获取该值时，可以通过引用DOM(比如getElementById('xxx'))去拿到相应的值。具体以input为例的话，假设我们有如下input:

<input id="text" type="text"/>

这段HTML代码会在网页上呈现出一个输入框，当用户输入内容后，该input输入框中的内容会自动映射到它的value属性上，我们可以通过document.getElementById('test').value获取到输入的内容。如下：

这种由DOM元素自身处理其状态的行为在React中被认为是非受控的，也就是说不受React控制。

三、React中的非受控组件

React中的非受控组件的道理和上面的HTML原生form元素的行为是一样的，不同的是，React组件中无需以类似于document.getElementById('test')这样的方式去拿到对应元素的引用，React中采用的通常采用的是ref的方式去达到同样的效果。如下：

class Form extends Component {
  handleSubmitClick = () => {
    const name = this._name.value;
    // do something with `name`
  }

  render() {
    return (
      <div>
        <input type="text" ref={input => this._name = input} />
        <button onClick={this.handleSubmitClick}>Sign up</button>
      </div>
    );
  }
}

采用ref也就意味着，通常我们需要将对该元素的引用存储在组件的属性上以备后续使用。也就是上述例子中的_name上。当我们在需要用到input的值(比如提交表单)的时候，我们可以用this._name.value的形式获取。

四、React中的受控组件

受控组件，顾名思意，就是说组件的状态受React控制，同样以input为例，作为受控组件，通常都需要给组件一个显示的value属性，该属性的值绑定为一个state。同时该组件还应该有一个函数，用于在恰当的时机来更新这个state。比如：

class Form extends Component {
  constructor() {
    super();
    this.state = {
      name: '',
    };
  }

  handleNameChange = (event) => {
    this.setState({ name: event.target.value });
  };

  render() {
    return (
      <div>
        <input
          type="text"
          value={this.state.name}
          onChange={this.handleNameChange}
        />
      </div>
    );
  }
}

input元素的value更新并不是由DOM自身来完成的，而是由React来完成的。具体的过程如下：

  • 最初，由于this.state.name为空串，所以呈现出来的是空输入框
  • 当你输入一个字符的时候，handleNameChange被触发，从而更改state。由于state改变，React组件会重新render，这样输入框中就有你输入的内容了。

这意味着，你的数据(state)和UI(input)始终都是同步的。state给value赋值，而输入的改变又会要求组件去改变state。

这也意味着，组件可以对输入进行即时响应，比如：

  1. 就地反馈，就像表单校验器一样。
  2. 禁用提交按钮，除非所有的输入都被验证通过。
  3. 强制某种特殊的输入，比如信用卡号等。

五、是什么让一个组件变成受控组件呢？

如果一个表单元素通过prop的方式设置它的值，那么它就成为了受控组件。不同的表单元素通过不同的属性设置它的值，下面就是对几种表单元素的总结：

Element Value property Change callback New value in the callback
<input type="text" /> value="string" onChange event.target.value
<input type="checkbox" /> checked={boolean} onChange event.target.checked
<input type="radio" /> checked={boolean} onChange event.target.checked
<textarea /> value="string" onChange event.target.value
<select /> value="option value" onChange event.target.value

六、结论

受控表单字段和非受控表单字段都有其优点，你要做的就是评估你的具体情况并选择恰当的方法 - 对你够用就行。如果你的表单在交互方面相当简单，那么结合ref使用非受控组件完全OK。你不用管有些文章说的非受控组件如何如何不好啥的。
