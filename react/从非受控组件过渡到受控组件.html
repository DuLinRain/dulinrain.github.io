从非受控组件过渡到受控组件

一、概述

在React之受控组件与非受控组件这篇文章中我们已经了解到了受控组件和非受控组件的区别。非受控组件在一些简单的场景中就能够满足我们的需求了，但是当我们遇到稍微复杂点的场景(比如，禁用提交按钮或立即验证表单域)的时候,你就需要使用受控组件了。那么问题来了，怎么样让你的非受控组件过渡到受控组件呢？

二、非受控组件 过渡到 受控组件

了解了非受控组件和受控组件的区别后，从非受控组件过渡到受控组件就是一件非常明确和容易的事儿了。我们只需要完成以下几个步鄹：

  1. 找到所有的表单控件，如input, textboxes, selects, checkboxes等
  2. 为这些空间初始化各自的state
  3. 让这些控件从步鄹2中的state获取他们的值
  4. 通过控件的onChange属性设置这些控件的值
  5. 在你的表单提交按钮的响应函数中，通过state而不是通过ref拿他们的值

假设我们有下面这样一个非受控表单组件：

class UncontrolledForm extends Component {
  handleSubmit = () => {
    const email = this._emailInput.value;
    const agreeCheckbox = this._agreeCheckbox.checked;
    ...
  };

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input type="email" ref={inp => this._emailInput = inp} />
        <input type="checkbox" ref={inp => this._agreeCheckbox = inp} />
      </form>
    );
  }
}

那么我们开始按照上面的几个步骤来将这个非受控组件改造成受控组件。

2.1 找到所有的表单控件

从上面例子来看，我们有两个表单输入控件，一个是email，一个是checkvox

2.2 为这些空间初始化各自的state

email控件的state初始化为空字符串，checkbox控件的state初始化为布尔false：

class ControlledForm extends Component {
  constructor() {
    super();
    this.state = {
      email: '',
      agree: false,
    };
  }
}

2.3 让这些控件从步鄹2中的state获取他们的值

也就是说给他们绑定state,如下：

class ControlledForm extends Component {
  // constructor omitted

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input type="email" value={this.state.email} />
        <input type="checkbox" checked={this.state.agree} />
      </form>
    );
  }
}

注意： 给email绑定在value上，给checkbox绑定在checked上。为什么这么做，可以参考React之受控组件与非受控组件

2.4 通过控件的onChange属性设置这些控件的值

class ControlledForm extends Component {
  // contructor omitted

  handleEmailChange = (evt) => {
    this.setState({ email: evt.target.value });
  };

  handleAgreeChange = (evt) => {
    this.setState({ agree: evt.target.checked });
  };

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input onChange={this.handleEmailChange} />
        <input type="email" value={this.state.email} onChange={this.handleEmailChange} />
        <input type="checkbox" checked={this.state.agree} onChange={this.handleAgreeChange} />
      </form>
    );
  }
}

2.5 在你的表单提交按钮的响应函数中，通过state而不是通过ref拿他们的值

我们在受控组件中是这么拿值的：

class ControlledForm extends Component { handleSubmit = () => { const email = this.state.email; const agree = this.state.agree; ... }; }

最终改造完后的受控组件的完整代码如下：

class ControlledForm extends Component {
  constructor() {
    super();
    this.state = {
      email: '',
      agree: false,
    };
  }

  handleSubmit = () => {
    const email = this.state.email;
    const agree = this.state.agree;
    ...
  };

  handleEmailChange = (evt) => {
    this.setState({ email: evt.target.value });
  };

  handleAgreeChange = (evt) => {
    this.setState({ agree: evt.target.checked });
  };

  render() {
    return (
      <form onSubmit={this.handleSubmit}>
        <input onChange={this.handleEmailChange} />
        <input type="email" value={this.state.email} onChange={this.handleEmailChange} />
        <input type="checkbox" checked={this.state.agree} onChange={this.handleAgreeChange} />
      </form>
    );
  }
}

至此，所有的改造都已经完成了。
