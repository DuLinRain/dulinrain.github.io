ES9提案之剩余运算符与展开运算符特性

剩余运算符/展开运算符是由Sebastian Markbåge提出的提案，它支持：

  • 剩余运算符可支持对象解构。当前，我们的剩余运算符只支持数组解构和参数定义。
  • 展开运算符可支持对象字面量。当前我们的展开运算符只支持数组字面量以及函数和方法调用。

一、剩余运算符在对象解构中的使用

在对象解构模式(参考ES6之解构)中,剩余运算符将所有可枚举自有属性解构出来放在它的操作数上，除已经明确解构出来的部分属性外。比如说下面的例子：

const obj = {foo: 1, bar: 2, baz: 3};
const {foo, ...rest} = obj;
// Same as:
// const foo = 1;
// const rest = {bar: 2, baz: 3};

如果你使用对象解构来处理命名参数，剩余运算符可以让你能够收集所有剩下的参数：

function func({param1, param2, ...rest}) { // rest operator
    console.log('All parameters: ',
        {param1, param2, ...rest}); // spread operator
    return param1 + param2;
}

语法限制

在每个对象字面量的顶层(最外层), 剩余运算符最多只能出现一次，并且必须出现在末尾：

const {...rest, foo} = obj; // 语法错误，只能放在末尾
const {foo, ...rest1, ...rest2} = obj; // 语法错误，只能出现一次

但是，在嵌套情况下（也就是不在一个层级）你可以多次使用剩余运算符：

const obj = {
    foo: {
        a: 1,
        b: 2,
        c: 3,
    },
    bar: 4,
    baz: 5,
};
const {foo: {a, ...rest1}, ...rest2} = obj;
// Same as:
// const a = 1;
// const rest1 = {b: 2, c: 3};
// const rest2 = {bar: 4, baz: 5};

三、对象展开运算符的常见使用案例

在这一小节，我们将看一看哪些地方可以使用展开运算符。我也会讲述如何使用Object.assign()做同样的事。（Object.assign()和展开运算符非常相似，我们接下来会详细比较他们）

3.1 克隆对象

克隆一个对象obj的可枚举自有属性：

const clone1 = {...obj};
const clone2 = Object.assign({}, obj);

克隆出来的对象(在这里指clone1和clone2)的原型都是Object.prototype，Object.prototype也是创建对象字面量的默认原型。

克隆一个对象obj, 包括它的原型：

const clone1 = {__proto__: Object.getPrototypeof(obj), ...obj};
const clone2 = Object.assign(Object.create(Object.getPrototypeof(obj)), obj)

注意：代码中的__proto__只是在部分浏览器中支持，在Nodejs中是不支持的。

3.2 真正地克隆对象

有些时候，你不只是需要克隆一个对象的可枚举自有属性，你还需要克隆它的一些特性（比如writable, enumerable等）以及getters 和 setters。这个时候，展开运算符和Object.assign都做不到了。这个时候你需要使用属性描述符(参考： ECMAScript2017之getOwnPropertyDescriptors):

const clone1 = Object.defineProperties({},
    Object.getOwnPropertyDescriptors(obj));

如果你还想保留原型，那么你需要用到Object.create():

const clone2 = Object.create(
    Object.getPrototypeOf(obj),
    Object.getOwnPropertyDescriptors(obj));

3.3 陷阱：克隆总是浅的

你一定要记住的是，我们现在所看到的这几种克隆方式都是浅克隆，你只拿到了浅拷贝。如果克隆的对象的某个属性还是一个对象，那么我们拿到的只是那个对象的引用，也就是指向同一个地方，它不会递归地、深度地克隆自己。

const original = { prop: {} };
const clone = Object.assign({}, original);

console.log(original.prop === clone.prop); // true
original.prop.foo = 'abc';
console.log(clone.prop.foo); // abc

3.4 其它使用案例

合并两个对象：

const merged = {...obj1, ...obj2};
const merged = Object.assign({}, obj1, obj2);

定义默认值：

const DEFAULTS = {foo: 'a', bar: 'b'};
const userData = {foo: 1};

const data = {...DEFAULTS, ...userData};
const data = Object.assign({}, DEFAULTS, userData);
    // {foo: 1, bar: 'b'}

非破坏性地更新对象的属性值：

const obj = {foo: 'a', bar: 'b'};
const obj2 = {...obj, foo: 1};
const obj2 = Object.assign({}, obj, {foo: 1});
    // {foo: 1, bar: 'b'}

内联地指定属性的默认值：

const userData = {foo: 1};
const data = {foo: 'a', bar: 'b', ...userData};
const data = Object.assign({}, {foo:'a', bar:'b'}, userData);
    // {foo: 1, bar: 'b'}

四、展开对象 VS Object.assign()

展开运算符和Object.assign()非常的相似，他们之间的唯一区别就是：展开运算符定义新的属性，而Object.assign()设置属性。这到底意味着什么我们接下来探讨。

4.1 使用Object.assign()的两种方式

有两种使用Object.assign()的方式。

第一， 破坏性的(现有的对象被改变)

Object.assign(target, source1, source2);

这里，target被修改，source1, source2被拷贝给target。

第二， 非破坏性的(任何现有的对象都不会改变)

const result = Object.assign({}, source1, source2);

这里，一个新的对象被通过对象字面量的方式创建，source1, source2被拷贝给它。最后这个新的对象被返回并赋值给result。

展开运算符和上面的第二种方式非常相似。 接下来我们来看看，他们两到底哪里相似，又有哪些差别。

4.2 展开运算符和Object.assign()都是通过get来读取属性值的

在将源对象的属性值写入目标对象之前，展开运算符和Object.assign()都是通过get来读取源对象的属性值。因此，在这个过程中，getters 被转换成了普通数据属性。

4.4 展开运算符和Object.assign()都只考虑可枚举自有属性

    inheritedEnumerable: 1,

const obj = Object.create(proto, {

        value: 2,
        enumerable: true,
    },
    ownNonEnumerable: {
        value: 3,
        enumerable: false,
