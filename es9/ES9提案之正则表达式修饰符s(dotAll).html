ES9提案之正则表达式修饰符s(dotAll)

这篇文章讲述由Mathias Bynens提出的正则表达式修饰符 s (dotAll)提案。

一、概述

我们都知道，JS正则表达式中点号(.)的意思是：

匹配除换行符 \n 之外的任何单字符。要匹配 . ，请使用 \. 。

也就是很明确的说了，它不能匹配一行的结束符：

/^.$/.test('\n')
false

二、传统正则表达式中点号(.)的限制

正则表达式中点号(.)有两大限制：

首先，它无法匹配星际象限(non-BMP, 即非BMP象限，关于JS字符集可参考JavaScript之一坨翔引发的血案)的字符，比如说emoji:

/^.$/.test('😀')
false

但是呢，这个可以用ES6新增的语法解决：

/^.$/u.test('😀')
true

其次就是它无法匹配行结束符：

/^.$/.test('\n')
false

目前有一些折中的方法可以绕过这个限制，达到预期的效果，比如[^] (“匹配除非字符外的所有字符”) 或 [\s\S] (“匹配空白符或非空白符”)：

2.1 ECMAScript的行结束符

ECMAScript中的行结束符会影响：

  • 所有正则表达式中不带s修饰符的点(.)符号
  • 使用了m修饰符的正则表达式中的开始（^）、结束（$）锚点符

下面这四种字符都被ECMAScript认为是行结束符：

  • U+000A LINE FEED (LF) (\n)
  • U+000D CARRIAGE RETURN (CR) (\r)
  • U+2028 LINE SEPARATOR
  • U+2029 PARAGRAPH SEPARATOR

也有一些新行字符不被ECMAScript认为是行结束符：

  • U+000B VERTICAL TAB (\v)
  • U+000C FORM FEED (\f)
  • U+0085 NEXT LINE

这三种符号可以被不带修饰符的点号匹配：

/^...$/.test('\v\f\u{0085}')
true

三、提案

该提案引入了一个新的修饰符s(singleline的简写)，使得点号(.)可以匹配到行结束符：

/^.$/s.test('\n')
true

它的完整名字是dotAll：

3.1 dotAll vs multiline

  • dotAll只影响点号(.)
  • multiline只影响开始（^）、结束（$）锚点符

四、常见问题

4.1 为什么修饰符的名字是/s？

dotAll很好地描述了这个修饰符所做的事情，所以，可以说，起个/a 或 /d可能会是个更好的名字，但是，/s已经是其它语言（Perl, Python, Java, C#, …）中存在的一个名字。
