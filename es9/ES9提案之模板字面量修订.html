ES9提案之模板字面量修订

由Tim Disney提交的模板字面量修订ECMAScript提案给予了标记模板字面量（tagged template literals）内部更加自由的语法.

一、标记函数和转义序列

有了标记模板字面量，你可以通过在一个模板字面量前面声明一个函数来实现函数调用：

String.raw`\u{4B}`
"\u{4B}"

这里，String.raw就是所谓的标记函数，标记函数接收模板字面量中固定字符串片段的两个版本：

  • Cooked： 转义序列被解释。\u{4B} 变成 'K'.
  • Raw： 转义序列就是普通文本。\u{4B} 变成 \\u{4B} .

下面的标记函数解释了这是如何做到的：

function tagFunc(tmplObj, substs) {
    return {
        Cooked: tmplObj,
        Raw: tmplObj.raw,
    };
}

二、问题：一些文本在反斜线后面会视为不合法

问题是，即使使用raw版本，ES2016中的模板字面量也没有完全的自由度。加上反斜线之后，很多字符序列都变得不再合法：

  • \u表示一个Unicode转义，它必须看起来像\u{1F4A4} 或 \u004B这样.
  • \x 表示16进制转义, 他必须看起来像 \x4B这样.
  • \ 跟上数字表示八进制序列 (比如 \141). 八进制转义在模板字面量以及严格模式下的字符字面量中是禁止的。

这些都会阻止像下面这样的标记模板字面量：

latex`\unicode`
windowsPath`C:\uuu\xxx\111`
